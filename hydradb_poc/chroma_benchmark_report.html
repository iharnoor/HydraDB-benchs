The Thesis: Pure Vector Search Has Fundamental Blind Spots

Cosine similarity computes a single number: how close two embeddings are in vector space. It has no understanding of negation, time, entity relationships, or geographic structure. HydraDB's hybrid approach (vector + knowledge graph + BM25 full-text) addresses several of these gaps. At the AI answer layer, the LLM compensates for some ranking issues — if the right chunk is anywhere in top-5, the answer can still be correct.

Negation Blindness
"NOT a birthday" matches birthday chunks highest
No Temporal Edges
Can't reason about "the day after" or chronological order
No Entity Direction
Can't distinguish Katie→Harnoor from Harnoor→Katie
No Geographic Awareness
"Outside the US" returns US-based events
Aggregation Ceiling
Top-k can't scan all chunks for exhaustive lists
LLM Compensates
AI layer masks retrieval differences when correct chunk is in top-5

Key Takeaway

Scored comparison reveals HydraDB's nuanced advantage

Binary YES/NO verdicts obscure cases where both systems answer partially. When scored on a 1–10 scale, HydraDB wins 3 of 5 head-to-head comparisons — excelling at semantic precision (milestone inference), negation handling (filtering out birthdays), and entity direction (cultural adoption details). Vector DB's single win comes from exhaustive listing, where similarity-based retrieval casts a wider net across celebration-type chunks. Both systems tie on straightforward extraction tasks.
