Key Takeaways

What these results mean for choosing a memory system

~

HydraDB: 2.4x Faster Retrieval

HydraDB averaged 1,069ms vs Mem0's 2,519ms. Mem0 needed retries on 6 of 11 tests (the ~3,600ms spikes in the chart), while HydraDB's /recall_preferences endpoint returned results consistently under 1.6s. For real-time applications like chatbots or dating profile assistants, this latency gap matters.

+

HydraDB: Multi-Chunk Retrieval Wins for Synthesis

HydraDB's biggest advantage was returning multiple related chunks in a single query. For the Miami timeline test, it returned both 2021 and 2022 trips. For relationship progression, it pulled the overview + Valentine's Day + early meetings. Mem0 typically returned a single summarized memory, which is cleaner but misses cross-referencing opportunities. When the question requires connecting dots across events, HydraDB's broader recall radius is decisive.

!

LLM + Retrieval: Better Together

The birthday test proved this: HydraDB's raw retrieval ranked Katie's birthday above Harnoor's, but with a Gemini summarization layer, the answer was correct. Retrieval provides the raw context; the LLM filters, disambiguates, and produces a precise one-line answer. Both systems benefit from this pattern — the retrieval engine doesn't need to be perfect, it just needs to include the right data somewhere in its results.

*

Both: Strong on Cultural/Semantic Queries

For questions about meaning ("Why was the 4th of July special?", "What was the significance of attending a wedding?"), both systems performed equally well. The cultural significance was explicitly stated in the source text, and both retrieval systems surfaced it correctly. This suggests that when semantic meaning is embedded in the writing itself, the retrieval method matters less than the quality of the source data.

Bottom Line

Testing with real relationship data (18 timeline memories about Harnoor & Katie), HydraDB won 5 of 11 tests, Mem0 won 0, and 6 were ties. HydraDB was 2.4x faster and excelled at multi-event synthesis, temporal queries (Miami trips, Princeton trip), and entity disambiguation when paired with an LLM layer. For a dating/relationship memory use case, HydraDB's ability to retrieve multiple related chunks makes it particularly well-suited — relationships are inherently about connecting events across time.
